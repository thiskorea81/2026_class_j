# 📊 정렬 알고리즘 단계별 상세 분석 (Sorting Simulation)

본 자료는 `10, 6, 2, 4, 3` 데이터를 활용하여 주요 정렬 알고리즘의 동작 과정을 비교 분석합니다.

---

## 1. 버블 정렬 (Bubble Sort)
**원리:** 인접한 두 항목을 비교하여 큰 값을 뒤로 보냅니다. 한 회전이 끝날 때마다 가장 큰 값이 맨 뒤에 고정됩니다.

### [데이터: 10, 6, 2, 4, 3]

| 단계 | 상세 분석 (비교 및 교환 과정) | 결과 | 비교/교환 |
| :--- | :--- | :--- | :---: |
| **1단계** | (10,6)교환 → (10,2)교환 → (10,4)교환 → (10,3)교환 | 6, 2, 4, 3, **[10]** | 4 / 4 |
| **2단계** | (6,2)교환 → (6,4)교환 → (6,3)교환 | 2, 4, 3, **[6], 10** | 3 / 3 |
| **3단계** | (2,4)유지 → (4,3)교환 | 2, 3, **[4], 6, 10** | 2 / 1 |
| **4단계** | (2,3)유지 | **[2], 3, 4, 6, 10** | 1 / 0 |

---

## 2. 삽입 정렬 (Insertion Sort)
**원리:** 두 번째 자료부터 시작하여 앞의 정렬된 부분과 비교하며 적절한 위치에 삽입합니다.

### [데이터: 10, 6, 2, 4, 3]

| 단계 | 상세 분석 (Key와 비교 대상) | 결과 | 비교/교환 |
| :--- | :--- | :--- | :---: |
| **1단계** | **Key: 6** / (10,6) 비교 → 10을 뒤로 밀고 6 삽입 | `[6, 10], 2, 4, 3` | 1 / 1 |
| **2단계** | **Key: 2** / (10,2), (6,2) 비교 → 10, 6 밀고 2 삽입 | `[2, 6, 10], 4, 3` | 2 / 2 |
| **3단계** | **Key: 4** / (10,4), (6,4) 비교 → 10, 6 밀고 4 삽입 | `[2, 4, 6, 10], 3` | 2 / 2 |
| **4단계** | **Key: 3** / (10,3), (6,3), (4,3) 비교 → 10, 6, 4 밀고 3 삽입 | `[2, 3, 4, 6, 10]` | 3 / 3 |

---

## 3. 퀵 정렬 (Quick Sort)
**원리:** 피봇(Pivot)을 기준으로 L(큰 수 탐색)과 R(작은 수 탐색)을 교환하며 분할합니다.

### [데이터: 10, 6, 2, 4, 3]

#### **Round 1: Pivot = 10**
- **탐색:** L은 10보다 큰 수 없음, R은 10보다 작은 `3` 탐색.
- **분석:** L과 R이 역전되었으므로 **Pivot(10)과 R(3) 교환**.
- **결과:** `3, 6, 2, 4, [10]` (**10 확정**) | 비교 4 / 교환 1

#### **Round 2: Pivot = 3 (대상: 3, 6, 2, 4)**
- **탐색1:** L은 `6`, R은 `2` 탐색 후 **L/R 교환** (`3, 2, 6, 4`).
- **탐색2:** 다시 탐색 시 L(6)과 R(2) 인덱스 역전 확인.
- **분석:** 역전되었으므로 **Pivot(3)과 R(2) 교환**.
- **결과:** `2, [3], 6, 4, 10` (**3 확정**) | 비교 3 / 교환 2

#### **Round 3: Pivot = 6 (대상: 6, 4)**
- **탐색:** L은 큰 수 없음, R은 작은 수 `4` 탐색. 인덱스 역전 발생.
- **분석:** **Pivot(6)과 R(4) 교환**.
- **결과:** `2, 3, 4, [6], 10` (**6 확정**) | 비교 1 / 교환 1
