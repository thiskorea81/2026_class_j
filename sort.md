# 📚 정렬 알고리즘 단계별 시각화 (Sorting Algorithms)

본 문서는 버블 정렬, 삽입 정렬, 퀵 정렬의 동작 과정을 학습자가 이해하기 쉽게 단계별로 시뮬레이션한 자료입니다.

---

## 1. 버블 정렬 (Bubble Sort)
**원리:** 인접한 두 원소를 비교하며 큰 값을 뒤로 보냅니다. 한 회전이 끝나면 가장 큰 값이 맨 뒤에 고정됩니다.

### 데이터: `10, 6, 2, 4, 3`

| 단계 | 비교/교환 | 정렬 과정 | 상태 |
| :--- | :---: | :--- | :--- |
| **초기값** | - | `10, 6, 2, 4, 3` | - |
| **1단계** | 4회 / 4회 | `6, 10, 2, 4, 3` → `6, 2, 10, 4, 3` → `6, 2, 4, 10, 3` → `6, 2, 4, 3, [10]` | **10** 확정 |
| **2단계** | 3회 / 3회 | `2, 6, 4, 3, 10` → `2, 4, 6, 3, 10` → `2, 4, 3, [6, 10]` | **6** 확정 |
| **3단계** | 2회 / 1회 | `2, 4, 3, 6, 10` → `2, [3, 4, 6, 10]` | **4** 확정 |
| **4단계** | 1회 / 0회 | `[2, 3, 4, 6, 10]` | **전체 완료** |

---

## 2. 삽입 정렬 (Insertion Sort)
**원리:** 두 번째 원소부터 시작(Key)하여, 앞부분의 정렬된 부분 중 적절한 위치를 찾아 삽입합니다.

### 데이터: `10, 6, 2, 4, 3`

* **1단계 (Key: 6)**
    * `10`과 `6` 비교 → `6`을 앞으로 삽입
    * 결과: `[6, 10], 2, 4, 3`
* **2단계 (Key: 2)**
    * `10`, `6`과 차례로 비교 → `2`를 맨 앞으로 삽입
    * 결과: `[2, 6, 10], 4, 3`
* **3단계 (Key: 4)**
    * `10`, `6`과 비교하여 그 사이에 삽입
    * 결과: `[2, 4, 6, 10], 3`
* **4단계 (Key: 3)**
    * `10`, `6`, `4`와 비교하여 `2`와 `4` 사이에 삽입
    * 결과: `[2, 3, 4, 6, 10]`

---

## 3. 퀵 정렬 (Quick Sort)
**원리:** 피봇(Pivot)을 기준으로 작은 값은 왼쪽, 큰 값은 오른쪽으로 분할하며 정렬합니다.

### 초기 데이터: `10, 6, 2, 4, 3`

#### [Step 1] 피봇: 10
1. **L(Left)**: 10보다 큰 수 찾기 → 없음 (오른쪽 끝 이동)
2. **R(Right)**: 10보다 작은 수 찾기 → `3`
3. **교환**: L과 R이 역전되었으므로 **Pivot(10)**과 **R(3)** 교환
   - **결과**: `3, 6, 2, 4, [10]` (**10 고정**)

#### [Step 2] 피봇: 3 (대상: 3, 6, 2, 4)
1. **L**: 3보다 큰 수 → `6`
2. **R**: 3보다 작은 수 → `2`
3. **교환**: L(6)과 R(2) 교환 → `3, 2, 6, 4`
4. **역전 발생**: 다시 탐색 시 L(6)과 R(2)의 인덱스가 역전됨
5. **교환**: **Pivot(3)**과 **R(2)** 교환
   - **결과**: `2, [3], 6, 4` (**3 고정**)

#### [Step 3] 분할 정복
* **왼쪽 (2)**: 요소가 하나이므로 자동 고정.
* **오른쪽 (6, 4) - 피봇: 6**:
    - L: 6보다 큰 수 없음 / R: 6보다 작은 수 `4`
    - 역전 발생으로 **Pivot(6)**과 **R(4)** 교환
    - **결과**: `4, [6]` (**6 고정**)
* **마지막 (4)**: 자동 고정.

### 🏁 최종 결과
> **2, 3, 4, 6, 10**
